# P3 Design: AI, Automation & Ecosystem

**Date**: 2026-03-07
**Phase**: 3 of 4
**Baseline**: P1 + P2 complete (Wave 1 data foundation + Wave 2 security & collaboration)
**Goal**: Add AI agents, event-driven automation, multi-chain identity, social linking, and broadcast publishing

---

## Architecture Overview

```
Indexer Webhook --> WebhookService --> EventEmitter2 --> TriggerMatcher
                                                              |
                                    +-------------------------+
                                    v                         v
                              WorkflowEngine            GasSponsorListener
                              (existing + new actions)   (balance check -> Enoki)
                                    |
                    +---------------+---------------+
                    v               v               v
              send_telegram   grant_discord_role  airdrop_token
              send_discord    issue_poap          ai_generate_content
                    |
                    +---> OAuth tokens from SocialLinkService (encrypted in vault)

AgentModule (LLM) --> Vercel AI SDK --> Claude / GPT (resolved per workspace)
    |                     |
    +- AnalystAgent       +- onFinish -> UsageLog (token count + cost)
    +- ContentAgent       +- API key resolution (platform / BYOK)
    +- ActionAgent        +- Rate limiter (Redis, per workspace)

BroadcastModule --> ChannelAdapterRegistry --> TelegramChannelAdapter
                                          --> DiscordChannelAdapter
                                          --> XChannelAdapter
                    |
                    +- Segment-aware: per-profile template variable substitution
                    +- Scheduling: cron-based delayed send

Multi-chain Identity
    +- Sui: existing SuiJsonRpcClient (gRPC)
    +- EVM: ethers.js direct RPC (ENS resolution)
    +- Solana: @bonfida/spl-name-service (SNS resolution)
    +- Balance/NFT aggregation: Moralis or Ankr API (multi-chain)
```

## Feature Specs

### P3-1: LLM Integration Foundation

**New module**: `packages/bff/src/agent/`

| Component | Responsibility |
|-----------|---------------|
| `AgentModule` | NestJS module, imports ConfigModule |
| `LlmClientService` | Vercel AI SDK wrapper, API key resolution (platform vs workspace BYOK) |
| `UsageTrackingService` | Log every LLM call: workspace, tokens, estimated cost |
| `RateLimitGuard` | Redis-backed per-workspace quota |
| `PromptTemplateService` | Load/render prompt templates |

**Prisma models**:

```prisma
model LlmUsageLog {
  id               String   @id @default(uuid())
  workspaceId      String   @map("workspace_id")
  userId           String   @map("user_id")
  agentType        String   @map("agent_type")
  model            String
  promptTokens     Int      @map("prompt_tokens")
  completionTokens Int      @map("completion_tokens")
  estimatedCostUsd Decimal  @map("estimated_cost_usd") @db.Decimal(10, 6)
  createdAt        DateTime @default(now()) @map("created_at")

  @@index([workspaceId, createdAt])
  @@map("llm_usage_logs")
}

model WorkspaceAiConfig {
  workspaceId       String  @id @map("workspace_id")
  provider          String  @default("anthropic")
  apiKeyEncrypted   String? @map("api_key_encrypted")
  monthlyQuotaUsd   Decimal @default(10) @map("monthly_quota_usd") @db.Decimal(10, 2)
  usedQuotaUsd      Decimal @default(0) @map("used_quota_usd") @db.Decimal(10, 2)
  quotaResetAt      DateTime @map("quota_reset_at")
  isEnabled         Boolean @default(true) @map("is_enabled")

  @@map("workspace_ai_configs")
}
```

**API key resolution flow**:
1. Check `WorkspaceAiConfig.apiKeyEncrypted` -> decrypt via Seal -> use as BYOK
2. Fallback to platform key from env `ANTHROPIC_API_KEY` / `OPENAI_API_KEY`
3. Rate limit check -> reject if over quota

**Billing model**: Hybrid
- Platform provides base quota (configurable per plan)
- BYOK option in workspace settings for unlimited usage
- Usage tracking for both modes

### P3-2: Journey Builder -- Event Triggers

**Event bus**: `@nestjs/event-emitter` (already installed in AppModule)

**Trigger types**:

| Trigger | Source | Real-time? |
|---------|--------|-----------|
| `wallet_connected` | Auth webhook | Yes (EventEmitter) |
| `nft_minted(collection)` | Indexer webhook | Yes (EventEmitter) |
| `token_transferred` | Indexer webhook | Yes (EventEmitter) |
| `defi_action(type)` | Indexer webhook | Yes (EventEmitter) |
| `time_elapsed(duration)` | Cron job | No (polling) |
| `segment_entered` | Segment refresh cron | No (polling) |
| `segment_exited` | Segment refresh cron | No (polling) |

**New components**:
- `TriggerMatcherService` -- listens to EventEmitter events, matches against active campaign triggers
- `CampaignTrigger` Prisma model -- campaignId, triggerType, triggerConfig (JSON)
- `SegmentDiffJob` -- cron job that detects segment membership changes and emits `segment_entered`/`segment_exited`
- `TimeTriggerJob` -- cron job for `time_elapsed` triggers
- Frontend: trigger node type selector in workflow canvas + condition node editor

**Condition nodes**: `if engagement_score > X`, `if has_tag('whale')`, `if balance > Y`

```prisma
model CampaignTrigger {
  id            String  @id @default(uuid())
  campaignId    String  @map("campaign_id")
  triggerType   String  @map("trigger_type")
  triggerConfig Json    @map("trigger_config")
  isEnabled     Boolean @default(true) @map("is_enabled")

  campaign Campaign @relation(fields: [campaignId], references: [id])

  @@index([triggerType, isEnabled])
  @@map("campaign_triggers")
}
```

**Future upgrade path**: Replace EventEmitter with BullMQ for retry, dead letter queue, horizontal scaling. Interface stays the same -- swap the transport layer only.

### P3-3: AI Analyst Agent

**Endpoint**: `POST /agents/analyst/query`

**Flow**:
1. User sends natural language query
2. Intent classification via LLM tool-use: `segment_query` | `report` | `trend_analysis`
3. LLM generates Prisma query DSL (not raw SQL -- safer, type-checked)
4. Execute query -> format results
5. Auto-generate chart config (score distribution, funnel, heatmap) as JSON
6. Frontend renders chart via existing chart library

**Safety**: LLM-generated queries restricted to read-only Prisma operations (`findMany`, `aggregate`, `groupBy`). No mutations.

### P3-4: Content Agent + Action Agent

**Content Agent** -- `POST /agents/content/generate`
- Input: segment description + channel (telegram/discord/email/x) + tone
- Output: marketing copy, customized per channel format
- Frontend: "AI Suggest Copy" button in campaign create + broadcast editor

**Action Agent** -- `POST /agents/action/plan` + `POST /agents/action/execute`
- Plan: NL -> structured action plan (target segment + actions + estimated cost/gas)
- Execute: human reviews plan -> confirms -> WorkflowEngine executes
- Safety: always requires human approval before execution

### P3-6: Social Linking (OAuth verify + auto-action)

**Platforms**:

| Platform | Auth method | Token storage | Action capability |
|----------|------------|---------------|-------------------|
| Discord | OAuth2 | Seal-encrypted in vault | grant_role, send_dm, post to channel |
| Telegram | Bot Login Widget | Seal-encrypted in vault | send_message (existing) |
| X (Twitter) | OAuth2 | Seal-encrypted in vault | post tweet, send_dm |
| Apple | ZkLogin via Enoki | N/A (wallet-based) | N/A |

```prisma
model SocialLink {
  id                  String   @id @default(uuid())
  profileId           String   @map("profile_id")
  platform            String
  platformUserId      String   @map("platform_user_id")
  platformUsername    String?  @map("platform_username")
  oauthTokenEncrypted String?  @map("oauth_token_encrypted")
  verified            Boolean  @default(false)
  linkedAt            DateTime @default(now()) @map("linked_at")

  profile Profile @relation(fields: [profileId], references: [id])

  @@unique([profileId, platform])
  @@index([platform, platformUserId])
  @@map("social_links")
}
```

**Frontend**: Profile detail -> Social tab with link/unlink per platform

### P3-7: Workflow Actions + Playbook Templates

**New actions** (register in WorkflowEngine action map):
- `grant_discord_role` -- uses verified Discord OAuth token from SocialLink
- `issue_poap` -- on-chain POAP/badge mint
- `ai_generate_content` -- calls ContentAgent inline during workflow

**Playbook templates**: Pre-built workflow JSON configs
- NFT Welcome: `nft_minted -> delay 1h -> send_telegram welcome -> grant_discord_role`
- DeFi Activation: `defi_action(swap) -> if balance > X -> airdrop_token`
- DAO Voting: `governance_vote -> send_discord thanks -> issue_poap`
- Membership Tier: `segment_entered(whale) -> grant_discord_role(VIP) -> send_telegram`

**API**: `GET /templates/playbooks` -> frontend template picker
**Campaign stats**: `GET /campaigns/:id/stats` -- entry count, per-node conversion, completion rate

### P3-8: Gas Station (Auto-sponsor)

**Trigger**: Event-driven via P3-2 event bus
- Listen for `wallet_connected` / `profile_created` events
- Check balance via Sui gRPC -> if < workspace threshold -> flag for sponsorship
- Next TX from that user -> Enoki sponsored mode (pay gas, not send SUI)

**Config** (workspace settings):
- `gasSponsorEnabled`: boolean toggle
- `gasSponsorThreshold`: minimum SUI balance (default: 0.1 SUI)
- `gasSponsorDailyLimit`: max sponsored TXs per day per workspace

## Design Decisions

1. **Vercel AI SDK over direct SDK**: Provider-agnostic, built-in token counting via `onFinish`, easy to swap models
2. **EventEmitter over BullMQ**: Simpler for current scale, can upgrade later (noted as future path)
3. **Hybrid real-time + cron triggers**: Real-time for indexer events, cron for time-based and segment membership changes
4. **Moralis/Ankr for balance aggregation**: Direct RPC only for name resolution (lightweight); balance + NFT portfolio via aggregator API
5. **Sui RPC is gRPC**: Use existing `SuiJsonRpcClient` for Sui balance, not the aggregator
6. **OAuth verify + auto-action for social**: All 4 platforms (Discord, Telegram, X, Apple), tokens encrypted via Seal
7. **Separate BroadcastModule**: Not shoehorned into Campaign -- different lifecycle (draft/schedule/send vs trigger/workflow)
8. **Human-in-the-loop for Action Agent**: Always requires approval before executing planned actions
