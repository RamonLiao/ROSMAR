## P2-1: Rust Indexer Completion

### Current State

Components exist but pipeline is **disconnected**:
- CheckpointConsumer: polls Sui RPC, extracts events (done)
- EventRouter: dispatch logic exists but **not wired** to consumer
- Handlers (NFT, DeFi, Audit): write to DB (done)
- Enricher (address -> profile_id): preloaded but **unused**
- AlertEngine (whale alerts): logic done but **not called** from pipeline
- GovernanceHandler: **does not exist**
- Webhook to BFF: AlertEngine has POST, but no general event webhook

### Target Architecture

```
Sui RPC (checkpoints)
  |
CheckpointConsumer.start()
  | events extracted
EventRouter.route()
  | dispatch by event type
+----------+----------+-------------+------------+
| NFT      | DeFi     | Governance  | Audit      |
| (exists) | (exists) | (NEW)       | (exists)   |
+----+-----+----+-----+------+------+-----+------+
     |          |             |            |
     v          v             v            v
  Enricher.resolve(address -> profile_id)
     |
     v
  BatchWriter.push(enriched_event)
     |
     +-> wallet_events INSERT (TimescaleDB)
     |
     +-> AlertEngine.check_and_alert()
     |     +-> webhook (whale-alert type)
     |
     +-> WebhookDispatcher.send()
           +-> POST /webhooks/indexer-event (all types)
```

```rust
GovernanceEvent {
    proposal_id: String,
    voter: String,
    vote_type: String,    // "for" | "against" | "abstain"
```

### BFF Webhook Receiver

```
POST /webhooks/indexer-event
  -> validate payload
  -> persist whale alerts to notification queue
  -> emit internal event (interface for future P3-2 Journey triggers)
  -> 200 OK
```

## P2-6: Seal SDK Integration

### Current State

- Web Crypto AES-GCM + PBKDF2 fully working (single-user encryption)
- Move `crm_vault::policy` has AccessPolicy with 3 rule types
- Move `Vault.seal_policy_id` field pre-reserved as `Option<ID>`
- Seal SDK: **not integrated at all**
- BFF vault service: Walrus upload/download working

### Design Decision

**Full Seal replacement** — no Web Crypto fallback.

Rationale:
- App is online-only (Sui RPC + Walrus + BFF required)
- Seal is foundation for P2-7 (ACL), P2-8 (Deal Room), P4-4 (GDPR)
- Two crypto paths = tech debt with no benefit
- Seal enables policy-based multi-user decryption (core vault value prop)

### UX: Minimize Wallet Signing

- Seal session key: sign once at workspace login, reuse for all encrypt/decrypt
- Current Web Crypto: sign per operation (worse UX)
- Session lifetime = workspace session (login -> logout)

### Encryption Flow Change

**Before (Web Crypto):**
```
sign message -> PBKDF2(signature) -> AES-GCM encrypt -> Walrus
  ^ every operation requires signing
  ^ single user only, no sharing
```

**After (Seal):**
```
login -> Seal session (sign once)
encrypt: Seal.encrypt(data, policyObjectId) -> blob -> Walrus
decrypt: Seal.decrypt(blob, policyObjectId) -> plaintext
  ^ session-based, no per-op signing
  ^ policy defines who can decrypt
```

### Data Flow

```
ENCRYPT:
  1. User picks/creates AccessPolicy (WORKSPACE_MEMBER default)
  2. Seal.encrypt(plaintext, policyObjectId) via session
  3. Upload encrypted blob to Walrus (get blobId)
  4. POST /vault/secrets { policyId, blobId, metadata }

DECRYPT:
  5. GET /vault/secrets/:profileId/:key -> { blobId, policyId }
  6. Download blob from Walrus
  7. Seal.decrypt(blob, policyObjectId) via session
     -> TEE verifies wallet in policy -> returns plaintext
```

### Code Changes

| Component | Change |
|-----------|--------|
| `lib/crypto/vault-crypto.ts` | DELETE (Web Crypto) |
| `lib/crypto/seal-crypto.ts` | NEW — Seal SDK wrapper |
| `useVaultCrypto` hook | Rewrite: policyId param, Seal session, no signMessage |
| `VaultItemForm` | Add policy selector (default: WORKSPACE_MEMBER) |
| `VaultItemCard` | Decrypt via Seal session (no sign prompt) |
| BFF `StoreSecretDto` | Add `sealPolicyId` field |
| BFF `VaultService` | Store policyId in metadata, remove crypto assumptions |
| Move `crm_vault` | Wire `set_blob()` with real Seal policy ID |
